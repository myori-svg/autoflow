## 1. 프로젝트 개요
---
### 한 줄 정의
AI 기반 자동 일정 배분 앱

### 제품 목표
사용자의 할일 관리 효율성을 극대화하고, 계획 수립에 대한 스트레스를 현저히 감소시켜, 복잡한 일정을 가진 사용자들이 모든 마감일을 성공적으로 지킬 수 있도록 돕습니다.

### 배경
많은 사용자가 여러 마감 기한이 겹치는 상황에서 일정을 관리할 때, 할일의 우선순위를 정하는 데 어려움을 겪고, 각 작업에 필요한 시간을 정확히 추정하지 못하며, 마감일에 임박해서야 급하게 처리하는 경향이 있습니다. 또한, 한 번 세운 계획이 유연하지 않아 변경사항에 대응하기 어렵다는 문제가 있습니다.

### 사용자 문제
사용자들은 여러 마감이 겹쳐 우선순위 설정에 어려움을 느끼고, 각 할일에 대한 정확한 소요 시간 예측에 실패하며, 이로 인해 마감일에 임박하여 급하게 작업을 처리하거나 계획이 틀어졌을 때 유연하게 대처하지 못하여 스트레스를 받습니다.

### 해결 방안
할일과 마감일만 입력하면 AI가 각 할일의 소요 시간을 예측하고, 사용자의 기존 스케줄과 선호도를 고려하여 주간 캘린더에 최적의 일정을 자동으로 배분해줍니다. 또한, 일정 변경 시 AI가 자동으로 재조정을 제안하여 유연한 계획 관리를 돕습니다.

### 차별점
기존의 디지털 캘린더나 노트/프로젝트 관리 도구는 사용자가 모든 일정을 수동으로 입력하고 배분해야 합니다. AutoFlow는 Gemini AI를 활용하여 할일의 소요 시간을 추정하고, 마감일 및 효율성을 고려하여 자동으로 일정을 배분해주는 독점적인 기능으로 사용자의 시간과 노력을 획기적으로 절감하며, 사용자 피드백을 통해 AI의 정확도를 지속적으로 향상시킵니다.

### 타겟 사용자
주로 여러 과제, 아르바이트, 동아리 활동 등으로 마감 기한이 자주 겹치는 대학생을 1차 타겟으로 합니다. 이들은 계획 수립 및 시간 관리에 어려움을 느끼며 효율성 증대와 스트레스 감소를 추구하는 사용자들입니다. 장기적으로는 복잡한 프로젝트와 여러 클라이언트 마감을 관리하는 프리랜서나 여러 업무가 동시다발적으로 발생하는 직장인들에게도 유용할 것입니다.

### 사용자 시나리오
대학생 김민준(22세) 씨는 이번 주에 팀 프로젝트 보고서 제출, 중간고사 공부, 아르바이트 근무, 동아리 회의 준비 등 다양한 마감 기한이 겹쳐 어떻게 시간을 배분해야 할지 막막합니다. 그는 AutoFlow 앱에 각 할일의 제목과 마감일을 입력합니다. AutoFlow는 AI를 통해 각 할일의 예상 소요 시간을 추정하고, 민준 씨의 기존 고정 일정(예: 수업 시간, 아르바이트 시간)을 고려하여 주간 캘린더에 가장 효율적인 스케줄을 자동으로 배분해줍니다. 민준 씨는 AI가 제안한 스케줄을 확인하고, 필요에 따라 일부 일정을 수동으로 조정한 후, 스마트 알림을 통해 계획대로 일정을 이행합니다. 할일을 마친 후에는 '딱 맞음', '오래 걸림', '일찍 끝냄' 버튼으로 회고를 기록하여 AI의 다음 스케줄링 정확도를 높이는 데 기여합니다.

### 핵심 지표
월간 활성 사용자(MAU) 20% 성장, 주간 할일 완료율 80% 달성, 사용자별 AI 예측 소요 시간과 실제 소요 시간의 오차율 10% 이내 유지, 사용자 만족도 설문조사에서 '계획 스트레스 감소' 항목 4점 이상(5점 만점), 회고 기능 참여율 50% 이상 달성.

### 리스크
AI의 소요 시간 예측 정확도가 낮을 경우 사용자 불만 증가, 경쟁 서비스의 유사 기능 출시, 개인 정보 보호 및 보안 문제 발생, 사용자 인터페이스/경험(UI/UX)이 복잡하거나 직관적이지 못할 경우 사용자 이탈.

### 검증 기준
- 할일 입력 후 AI 소요시간 추정 결과가 3초 내 반환되어야 한다
- 추정된 소요시간을 기반으로 마감일 이전에 일정이 배분되어야 한다
- 배분된 일정이 FullCalendar에 정상 표시되어야 한다

### 제약과 예외
- Gemini API 호출 실패 시 사용자에게 오류 메시지 표시, 수동 입력 허용
- MongoDB Atlas 연결 실패 시 서버 에러 응답 반환
- 마감일이 현재 시각보다 이전인 경우 입력 거부
