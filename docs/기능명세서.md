## 1. 할일 입력 및 마감일 설정
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 할일의 제목, 마감일, 그리고 선택적으로 중요도, 상세 설명을 입력할 수 있는 직관적인 UI를 제공합니다.

**수용 기준:**
```
1. ○ 할일 제목 (필수) 입력 필드가 제공되어야 한다.
2. ○ 마감일 (필수)을 선택할 수 있는 캘린더 피커가 제공되어야 한다.
3. ○ 할일의 중요도를 '높음', '중간', '낮음' 등으로 설정할 수 있는 옵션이 제공되어야 한다.
4. ○ 할일에 대한 추가 설명을 입력할 수 있는 텍스트 영역이 제공되어야 한다.
```


### 1.1 할일 제목 입력 필드
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

사용자가 할일의 제목을 입력할 수 있는 필수 입력 필드를 제공합니다. 이 필드는 할일을 식별하는 기본 정보입니다.


#### 1.1.1 할일 제목 입력 필드 UI 구현

사용자가 할일 제목을 입력하는 텍스트 입력 필드. 최대 100자 제한, 필수 항목임을 나타내는 시각적 표시(*) 포함, 입력 시 실시간 유효성 검사(빈 값 방지)를 수행합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 1.1.2 할일 제목 입력값 저장

사용자가 입력한 할일 제목을 임시 메모리 또는 로컬 스토리지에 저장하고, 폼 제출 시 데이터베이스에 저장합니다. 입력 중 자동 저장 기능 포함.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 1.1.3 할일 제목 입력 제한 및 검증

제목 길이 제한(최대 100자), 특수문자 허용 여부 확인, 중복 제목 생성 시 경고 메시지 제공.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


### 1.2 마감일 캘린더 피커
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

사용자가 할일의 마감일을 선택할 수 있는 인터랙티브 캘린더 피커를 제공합니다. 필수 입력 필드로 운영됩니다.


#### 1.2.1 캘린더 피커 컴포넌트 구현

인터랙티브 캘린더 UI로, 사용자가 마감일을 선택할 수 있습니다. 월/년도 네비게이션, 오늘 날짜 강조, 선택된 날짜 시각적 표시 포함.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 1.2.2 마감일 선택 저장

사용자가 캘린더에서 선택한 마감일(날짜)을 시스템에 저장하고, 선택 완료 후 캘린더 피커를 닫으며 선택된 날짜를 UI에 표시합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 1.2.3 마감일 유효성 검사

선택된 마감일이 현재 날짜 이후인지 검증합니다. 과거 날짜 선택 시 경고 메시지 표시. 필수 입력 확인으로 빈 값 방지.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


### 1.3 중요도 선택 옵션
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

할일의 중요도를 '높음', '중간', '낮음' 중에서 선택할 수 있는 드롭다운 또는 라디오 버튼 옵션을 제공합니다.


#### 1.3.1 중요도 드롭다운 UI

'높음', '중간', '낮음' 3개 옵션을 제공하는 드롭다운 메뉴. 선택되지 않은 초기 상태에서 '선택 안 함' 또는 기본값 표시.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 1.3.2 중요도 선택값 저장

사용자가 선택한 중요도 값('높음'=3, '중간'=2, '낮음'=1)을 데이터베이스에 저장합니다. 미선택 시 기본값(예: '중간') 설정.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 1.3.3 중요도 시각적 표시

선택된 중요도 레벨을 색상(높음=빨강, 중간=주황, 낮음=회색) 또는 아이콘(별 개수)으로 시각적으로 표시합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


### 1.4 상세 설명 텍스트 영역

할일에 대한 추가 설명, 노트, 세부 사항을 입력할 수 있는 텍스트 영역을 제공합니다.


#### 1.4.1 상세 설명 텍스트 영역 UI

#### 1.4.2 상세 설명 내용 저장

사용자가 입력한 상세 설명 텍스트를 데이터베이스에 저장합니다. 빈 값 허용(선택사항). 개행, 공백 포함된 원본 텍스트 그대로 저장.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

사용자가 상세 설명에 기본 마크다운 포맷(굵게, 이탤릭, 리스트)을 사용할 수 있도록 지원. 미리보기 기능 선택사항.
> **중요도**: 🟢 낮음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

## 2. AI 기반 일정 자동 배분
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 입력한 할일과 마감일을 기반으로 Gemini AI가 각 할일의 예상 소요 시간을 추정하고, 기존 일정 및 사용자의 선호도를 고려하여 가장 효율적인 주간 일정을 자동으로 생성하고 배분합니다.

**수용 기준:**
```
1. ○ 할일 입력 후 '자동 배분' 버튼 클릭 시 AI가 예상 소요 시간을 추정하여 각 할일에 할당해야 한다.
2. ○ AI는 마감일이 임박한 할일을 우선적으로 배분해야 한다.
3. ○ AI는 번아웃 방지를 위해 각 작업 사이에 충분한 휴식 시간(버퍼 타임)을 확보하여 일정을 배분해야 한다.
4. ○ AI는 각 태스크의 예상 시간에 따라 일정을 효율적으로 계획하여 캘린더에 표시해야 한다.
```


### 2.1 AI 소요 시간 추정 엔진
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

사용자가 입력한 할일의 제목과 설명을 분석하여 Gemini AI를 통해 각 할일의 예상 소요 시간을 자동으로 추정합니다.


#### 2.1.1 AI 소요 시간 추정 API 연동

사용자 입력 데이터(할일 제목, 설명, 중요도, 마감일)를 Gemini AI API로 전송하고, 예상 소요 시간(시간 단위)을 응답받는 API 연동을 구현합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 2.1.2 AI 시간 추정 알고리즘

사용자의 과거 회고 데이터(땁 맞음/오래 걸림/일찍 끝냄)를 학습하여 예측 모델을 최적화. 초기 사용자는 일반 통계 기반 추정값 반환.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 2.1.3 AI 추정 시간 캐싱 및 오류 처리

동일한 입력에 대한 중복 API 호출을 방지하기 위해 캐싱 구현. API 오류 시 폴백 로직(기본값 반환) 적용.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


### 2.2 자동 배분 버튼
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

'자동 배분' 버튼을 클릭하면 AI가 모든 할일의 소요 시간을 추정하고 캘린더에 배분하는 기능을 제공합니다.


#### 2.2.1 자동 배분 버튼 UI

할일 입력 폼 하단에 '자동 배분' 버튼을 배치. 필수 입력 필드(제목, 마감일) 완료 시에만 활성화, 로딩 중 상태 표시(스피너 또는 프로그레스 바).
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 2.2.2 자동 배분 실행 로직

버튼 클릭 시 입력된 모든 할일에 대해 AI 추정 시간을 일괄 요청하고, 이를 반환받아 각 할일에 저장합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 2.2.3 자동 배분 완료 피드백

배분 완료 후 성공 메시지와 함께 캘린더 뷰로 자동 전환. 사용자에게 생성된 일정 수, 총 소요 시간 요약 정보 제시.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


### 2.3 마감일 우선순위 로직
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

AI가 일정을 배분할 때 마감일이 임박한 할일을 우선적으로 배치하는 알고리즘을 구현합니다.


#### 2.3.1 마감일 우선순위 계산 로직

각 할일의 마감일까지 남은 일수를 계산(deadline - today). 일수가 적을수록 우선순위 점수 높음. 우선순위 점수 = 1 / (남은 일수 + 1).
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 2.3.2 마감일 기반 일정 배치

계산된 우선순위 점수에 따라 할일을 정렬하고, 주간 캘린더의 시간대에 우선순위 높은 순서대로 배치합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 2.3.3 마감일 근처 경고 시각화

마감일이 3일 이내인 할일을 빨간색 배경 또는 경고 아이콘으로 시각화. 1일 이내는 극도로 강조.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


### 2.4 휴식 시간 버퍼 최적화
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

번아웃을 방지하기 위해 AI가 각 작업 사이에 충분한 휴식 시간(버퍼 타임)을 자동으로 확보하여 일정을 배분합니다.


#### 2.4.1 휴식 시간 버퍼 정의

각 할일 사이에 최소 15분(조정 가능)의 휴식 시간을 확보. 하루 8시간 이상 연속 작업 방지. 점심시간(12:00-13:00) 자동 예약.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 2.4.2 휴식 시간 자동 삽입

AI 배분 알고리즘에서 할일 간 휴식 시간을 자동으로 삽입. 배분 결과에 '휴식' 블록으로 시각화.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 2.4.3 번아웃 방지 학습

사용자의 회고 데이터(오래 걸림)가 많은 시간대를 분석하여, 미래 배분 시 해당 시간대의 작업량을 줄이는 학습 로직.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

## 3. 주간 캘린더 시각화 및 수동 조정
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

AI가 배분한 일정을 주간 단위 캘린더 뷰로 명확하게 시각화하여 보여주며, 사용자가 필요에 따라 일정을 드래그 앤 드롭 방식으로 쉽게 변경하거나 수정할 수 있도록 합니다.

**수용 기준:**
```
1. ○ 배분된 모든 할일이 주간 캘린더에 시간대별로 명확하게 표시되어야 한다.
2. ○ 사용자는 캘린더 상에서 할일 블록을 드래그 앤 드롭하여 시작 및 종료 시간을 변경할 수 있어야 한다.
3. ○ 사용자는 할일 블록을 클릭하여 세부 정보를 확인하고, 제목, 설명, 마감일 등을 직접 수정할 수 있어야 한다.
4. ○ 수동으로 일정을 조정한 후, 변경된 내용이 즉시 캘린더에 반영되어야 한다.
```


### 3.1 주간 캘린더 뷰
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

AI가 배분한 모든 할일을 주간 단위로 시간대별로 명확하게 시각화하여 표시하는 캘린더 뷰를 제공합니다.


#### 3.1.1 주간 캘린더 레이아웃 구현

월요일~일요일 7개 칼럼, 시간축(00:00~23:59) 행으로 표현. 그리드 기반 레이아웃, 반응형 디자인 지원. 현재 주 기본 표시.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 3.1.2 주간 네비게이션

이전주/다음주 버튼, 오늘로 이동 버튼. 주간 날짜 범위 표시(예: 2024.01.01 - 2024.01.07).
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

## 7. AI 기반 하위 태스크 자동 생성

```

8. ○ 일정 블록으로 반영을 선택한 경우, 하위 태스크는 사용자가 선택적으로 일정 블록으로 생성되어 구글 캘린더에 반영되어야 하며, 상위 일정과의 연결 관계를 확인할 수 있어야 한다.
```

## 8. 외부 캘린더(iCal URL/구글 캘린더) 연동
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자의 주력 캘린더(우선: 구글 캘린더)와 AutoFlow를 양방향으로 연동하여, 기존 일정(수업/근무/약속 등)을 가져와 '이미 잡힌 시간'으로 반영하고, AutoFlow가 배분한 일정도 구글 캘린더에 자동으로 동기화합니다. iCal URL은 2차 확장(읽기 전용 가져오기) 옵션으로 둡니다.

**수용 기준:**
```
1. ○ 사용자는 구글 계정을 OAuth로 연결하여 연동을 시작할 수 있어야 한다.
2. ○ 사용자는 연동할 구글 캘린더(하나 이상)를 선택할 수 있어야 한다.
3. ○ AutoFlow는 선택된 구글 캘린더의 이벤트를 가져와 주간 캘린더 뷰에 표시해야 한다.
4. ○ 가져온 구글 캘린더 이벤트는 AI 자동 배분 시 '이미 잡힌 시간'으로 반영되어, 해당 시간대에는 일정이 배분되지 않아야 한다.
5. ○ AutoFlow에서 생성된 일정(할일 블록)은 구글 캘린더에 자동으로 생성되어야 한다.
6. ○ AutoFlow에서 일정 시간이 변경되거나 삭제되면, 구글 캘린더에서도 자동으로 수정/삭제되어야 한다.
7. ○ 동기화는 중복 생성을 방지해야 하며(AutoFlow가 만든 이벤트 식별자 유지), 재시도 시에도 동일 이벤트가 복제되지 않아야 한다.
8. ○ 사용자는 '구글 캘린더에서 수정' 또는 'AutoFlow에서 수정' 중 어떤 쪽을 우선으로 볼지(수정 우선권/충돌 정책)를 선택할 수 있어야 한다.
9. ○ 권한 만료/연동 오류가 발생하면 사용자에게 알리고 재연결을 유도해야 한다.
10. ○ 연동 해제 시, 사용자는 AutoFlow가 구글 캘린더에 만든 이벤트를 유지하거나 삭제할지 선택할 수 있어야 하며, 가져온 이벤트 표시는 중단되어야 한다.
11. ○ iCal URL은 2차 기능으로, 사용자는 iCal URL을 입력해 이벤트를 읽기 전용으로 가져올 수 있어야 한다.
```


### 8.1 구글 캘린더 OAuth 연결/재연결
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

구글 계정 OAuth로 연결하고, 권한 만료/오류 시 재연결을 지원합니다.


### 8.2 연동 캘린더 선택 관리
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

사용자가 동기화 대상 구글 캘린더를 1개 이상 선택/변경할 수 있게 합니다.


### 8.3 구글 캘린더 이벤트 가져오기/표시
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

선택된 캘린더의 이벤트를 가져와 주간 캘린더 뷰에 표시하고, '이미 잡힌 시간'으로 자동 배분에 반영합니다.


### 8.4 AutoFlow 일정 자동 동기화(생성/수정/삭제)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

AutoFlow에서 만든 일정이 구글 캘린더에 자동 반영되며, AutoFlow에서 변경/삭제하면 구글에서도 동일하게 반영됩니다. 중복 생성 방지 식별자를 유지합니다.


#### 8.4.1 동기화 대상 이벤트 식별자 설계

AutoFlow가 구글 캘린더에 만든 이벤트를 안정적으로 식별하기 위한 키를 정의합니다(예: AutoFlow 내부 일정 ID를 기반으로 한 고유 키). 재시도/재연결/중복 생성 상황에서도 동일 이벤트를 찾아 업데이트할 수 있어야 합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 8.4.2 구글 캘린더 이벤트 생성 동기화

AutoFlow에서 일정(할일 블록)이 생성되면 구글 캘린더에 이벤트를 자동 생성합니다. 생성 성공 시 구글 이벤트 ID를 저장하고, 실패 시 재시도 큐에 적재합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 8.4.3 중복 생성 방지 및 재시도 전략

네트워크 오류/타임아웃/중복 요청 시에도 동일 이벤트가 여러 번 생성되지 않도록 방지합니다. 재시도 횟수, 백오프(점점 느리게 재시도), 실패 시 사용자에게 상태 표시(동기화 대기/실패)를 정의합니다.

### 8.5 동기화 충돌 정책/수정 우선권 설정
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

구글 캘린더에서 수정 vs AutoFlow에서 수정 중 어느 쪽을 우선으로 볼지 사용자가 정책을 선택하고, 충돌 시 처리 규칙을 적용합니다.


#### 8.5.1 충돌 정책 옵션 정의

사용자가 선택 가능한 충돌 정책을 정의합니다(예: AutoFlow 우선 / 구글 캘린더 우선 / 충돌 시 확인 요청). 각 옵션의 동작(어떤 변경을 덮어쓰는지)을 명확히 규정합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 8.5.2 수정 우선권 설정 UI

설정 화면에서 ‘구글 캘린더에서 수정’ vs ‘AutoFlow에서 수정’ 우선권을 선택할 수 있는 UI를 제공합니다. 선택값은 즉시 저장되고, 동기화 로직에 적용되어야 합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 8.5.3 충돌 감지 기준

동일 이벤트에 대해 양쪽에서 변경이 발생했는지 판단하는 기준을 정의합니다(예: 마지막 수정 시각 비교, 필드 단위 변경 추적). 충돌로 판단되면 선택된 정책에 따라 처리됩니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 8.5.4 충돌 발생 시 사용자 안내 및 해결 흐름

정책이 ‘충돌 시 확인 요청’인 경우, 충돌 내용을 보여주고(어떤 필드가 어떻게 달라졌는지) 사용자가 ‘구글 값 적용’/‘AutoFlow 값 적용’을 선택하도록 합니다. 선택 결과가 즉시 동기화되어야 합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web
